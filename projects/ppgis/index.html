Application Development for Public Input

One of the crucial steps in large urban planning projects such as comprehensive plan updates is the soliciting and integration of public input. There are many ways to go about this- however, I wanted to develop a solution that was simple and easy for everyone to use. After teaching myself some basic HTML, CSS, and Javascript, I created a publicly available app to collect comments from interested members of the public. The basic idea behind this application was for it to be something lightweight that could be deployed anywhere, easily refitted to any jurisdiction, and allow the addition of comments organized into topics by simply double clicking on the map. I did not want a complex back-end, and this use of the ArcGIS API for Javascript allows secure data addition without access to any private data or the need for extra privileges. There is an additional script monitoring the incoming comments hourly for undesirable language. The code for this application is viewable on my Github.

Skills used: front-end development, Javascript, HTML, real-time data monitoring and quality assurance, app development, PPGIS (public participation GIS).
